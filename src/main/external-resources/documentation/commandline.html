Command line arguments

Digital Media Server can be launched with command line arguments. These arguments can either be used from the command line directly, from scripts or from shortcuts. The Digital Media Server (Select Profile) shortcut made by the Digital Media Server Windows installer is an example of this (launched with the profiles argument). DMS will be used as an abbreviation for Digital Media Server in the rest of this document.

Arguments

  • -p, --profile=<path> – Starts DMS with the profile from the file or folder specified in <path>. This is mostly useful for starting a specific profile from a script or shortcut.
  • -P, --profiles – Launches the Profile Chooser during DMS startup, allowing you to choose a profile file or folder for DMS to use.
  • -v, --trace – Starts DMS in "forced trace" mode, where the log level is forced to <trace> level regardless of the log level specified in the profile.
  • -c, --headless – Starts DMS without a GUI. It can be useful for running DMS in the background without any user interface, but be aware that under Windows there is no other way than using the task manager to stop DMS again. This option in most useful under Linux, where you can stop DMS again simply by pressing Ctrl + c. If no graphical environment is available, DMS will automatically start in this mode.
  • -C, --noconsole – Causes DMS to fail to start if the GUI can't be initialized.
  • -s, --scrollbars – Creates horizontal and vertical scroll bars in DMS' main windows, allowing the window to be smaller than DMS is designed for. It can be useful on low resolution desktops.
  • -db, --database
    • log, trace – Enables database logging.
    • downgrade – Deletes and recreates any database tables of a newer version. The data in the incompatible tables will be lost. This is useful if you have previously run a newer version of DMS that has upgraded one or more database tables to a version unknown to this version of DMS.
    • backup[=<name>] – Makes a copy of the database before downgrading it if any database tables are of a newer version. If <name> isn't provided, a name for the backup will be generated. This is useful if you have previously run a newer version of DMS that has upgraded one or more database tables to a version unknown to this version of DMS.
    • rename[=<name>] – Renames or moves the database and creates a new, empty database if there is a problem with the current database. If <name> isn't provided, a new name will be generated. This is useful if you have database errors that prevents DMS from starting but don't want to delete the database.
  • -V, --version – Displays the DMS version and exits.
  • -V, --version – Displays the command line options and exits.
To be able to use the command line arguments on Windows, follow this syntax:

Windows

DMS.exe argument 1 argument 2 ... argument X

Linux

./DMS.sh argument 1 argument 2 ... argument X

macOS

open -a "Digital Media Server" --args argument1 argument2 ... argument X
